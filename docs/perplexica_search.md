# Інтеграція з Perplexica (v1.12.1+)

Цей документ описує особливості роботи інструменту `web_search` з локальним сервером Perplexica.

## Особливості версії 1.12.1

Починаючи з версії 1.12.1, API Perplexica зазнало суттєвих змін. Основні відмінності:

1.  **Обов'язкові моделі**: Кожен запит на пошук повинен містити об'єкти `chatModel` та `embeddingModel` з вказанням `providerId` та `key`. 
2.  **Джерела замість режимів**: Параметр `focusMode` застарів. Тепер використовується масив `sources` (наприклад, `["web"]`).
3.  **Ендпоінти**: 
    - Пошук: `POST /api/search`
    - Список провайдерів: `GET /api/providers` (замість `/api/models`)
    - Конфігурація: `GET /api/config`

## Налаштування в розширенні

Для роботи пошуку необхідно вказати URL вашого сервера в налаштуваннях VS Code:
`openaistudio.perplexicaUrl`: наприклад, `http://10.1.0.138:3030`

## Процес пошуку та таймаути

Пошук через Perplexica — це багатоетапний процес:
1.  **Пошук у SearXNG**: Отримання результатів з пошукових систем.
2.  **Обробка результатів**: Очищення та підготовка тексту.
3.  **Генерація відповіді**: Використання LLM (наприклад, через Ollama) для синтезу відповіді.

Через складність процесу, особливо при використанні локальних моделей, запит може тривати **від 30 до 90 секунд**. В інструменті встановлено збільшений таймаут для запобігання передчасного обриву з'єднання.

## Діагностика та тестування

Для перевірки працездатності вашого сервера в проекті є допоміжні скрипти:

- `src/get_perplexica_info.ts`: Отримує список доступних провайдерів та моделей з вашого сервера.
- `src/test_perplexica.ts`: Тестує безпосередньо API запит з правильними параметрами.
- `src/test_service_call.ts`: Тестує виклик через внутрішній сервіс розширення.

Запуск тестів:
```bash
npx ts-node src/test_perplexica.ts
```

## Типові помилки

| Помилка | Значення | Що перевірити |
| :--- | :--- | :--- |
| **400 Bad Request** | Невірний формат JSON або відсутні обов'язкові поля. | Переконайтеся, що передаються `sources` та моделі з ID. |
| **500 Internal Error** | Помилка на самому сервері Perplexica. | Перевірте логи (`docker logs`). Часто це через недоступність SearXNG. |
| **Timeout Error** | Сервер не встиг відповісти за відведений час. | Перевірте навантаження на систему та швидкість роботи Ollama. |
